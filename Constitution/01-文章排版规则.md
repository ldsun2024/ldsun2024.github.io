# 文章 Markdown 排版规则

## 1. Front Matter（元数据）

每篇 md 文件以 `---` 包裹的 YAML 开头：

```yaml
---
title: "标题"
date: 2026-06-30T20:00:00+08:00
lastmod: 2026-06-30T20:00:00+08:00
tags:
  - 标签1
  - 标签2
---
```

- `date` 为编写文章时的实际当前时间（精确到分钟），同目录下每篇间隔 2 分钟
- `lastmod` 与 `date` 保持一致，后续修改文章时更新为修改时间
- `tags` 每篇文章至少 1 个标签

## 2. 章节标题格式

```
## N. 标题文字
```

- 使用 `##` 二级标题
- 编号后跟 `.` + 空格
- 标题前后各空一行

### 子标题格式

使用 `###` 三级标题，**不加编号**，作为 `## N.` 章节下的细分内容：

```
## 2. 稀缺心态，让人变笨

正文内容...

### 急于求成，焦虑丛生

子章节正文...

## 3. 单一视角
```

示例：

```
## 1. 千万别认定自己不够格

正文内容...

## 2. 找到朋友与自我

正文内容...
```

## 3. 段落

- 每段 2~4 句
- 段落之间用空行分隔
- 每段文字连贯，不截断换行

```
第一句。第二句。第三句。第四句。

下一段第一句。第二句。第三句。
```

## 4. 禁止行为

- ❌ 段落中间插空行断句
- ❌ 标题没有 `##` 前缀
- ❌ 使用 `##` 但不带编号（子标题应使用 `###`，而非无编号的 `##`）
- ❌ 使用 `·`（中文圆点）冒充列表标记
- ❌ 系列列举项（"第一帖/第二帖…""一是/二是…"等）连写在同一段落，未做结构化分割
- ❌ 标题与正文连在同一行
- ❌ 标题前缺少空行
- ❌ 禁止出现章节总标题（如 `## 第五章 学习力...`），章节名已由 front matter `title` 展示
- ❌ `## ` 必须在行首，禁止与上文在同一行（`\n\n##` 也不行，必须是独立行）
- ❌ 同一标题重复出现（如第4章曾有21节含7个空壳）
- ❌ 文件开头/段落开头残留提取痕迹（如 `]`、`——编者注` 等拼接文字）
- ❌ 正文中出现水印、广告文字（如"免费书享分更多搜索@雅书"）
- ❌ 保留无对应图片的引用标记（如 `（见图X-X）`、`如图X-X所示`、`图X-X` 等，因本书无插图，全部删除）
- ❌ 脚注与正文连在同一行，应用 `> 编者注：` 引用格式独立展示
- 正文中出现的脚注/编者注统一使用块引用格式：

```
> 编者注：注释内容。
```

## 5. 列表格式

### 无序列表

使用 `- `（短横线 + 空格）创建无序列表，每条**独立成行**：

```markdown
- 地面的小花让人仿佛置身于世外桃源
- 干净的天空作背景使车体更加突出
- 车斗和前轮变大，车身高于背后的山脉
```

- ❌ 禁止使用 `·`（中文圆点 `U+00B7`）冒充列表标记
- ❌ 禁止将多条列表项连写在同一个段落中

### 编号列表

使用 `1. ` 或 `（1）` 格式，每条**独立成段**：

```markdown
1. 第一项说明。具体内容...
2. 第二项说明。具体内容...
```

或

```
（1）第一项说明。具体内容...

（2）第二项说明。具体内容...
```

- ❌ 禁止多条编号挤在同一段落
- ❌ 禁止编号嵌在段落中间

### 系列列举项格式

凡系列性的知识条目（如"第一帖/第二帖/第三帖…""一是/二是/三是…""第一步/第二步……"），应使用以下两种格式之一结构化：

**格式 A — 有序列表（推荐用于纯列举）：**

```markdown
一是善学习。有些时候我们之所以看不到一些角度，是因为自身学识不够…

二是寻帮助。不要一个人闷头苦想，要学会主动寻求外部帮助…

三是多运动。适当的有氧运动能提升多巴胺水平…
```

**格式 B — 加粗标题 + 段落（推荐用于每项有较多解释内容）：**

```markdown
**第一帖，保持环境觉知，理智选择。** 对于有些人来说，受影响最大的就是格局和远见…

**第二帖，保持目标觉知，少即是多。** 知道自己的人生目标之所以最为重要…

**第三帖，保持欲望觉知，审视决策。** 当前最大的压力莫过于过多欲望对心智带宽的冲击…
```

- ❌ 禁止将系列列举项连写在同一段落中，不作结构化分割

## 6. 文件末尾

- 每篇文章结尾必须是**完整句子**，以句号 `。` 结束
- ❌ 禁止末尾出现截断文字或残缺的脚注标记（如 `[2`、`从熟悉`）
- ❌ 禁止末尾残留提取时的拼接痕迹
- 如果原文被截断，直接删除不完整的最后一句话，或补全为完整句子
